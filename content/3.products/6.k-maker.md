# K-Maker
_K-Maker is KurrawongAI's vocabulary management solution — a purpose-built platform that governs your organisation's reference data as a standards-compliant Knowledge Graph, with self-service interfaces so your teams can focus on content, not management._

---
## Challenges for enterprise reference data
Organisations typically manage dozens of reference datasets — product classifications, status codes, geographic hierarchies, industry standards — spread across siloed spreadsheets, duplicated across systems, and updated without governance or traceability. The result is inconsistent reporting, integration rework, and an inability to share reference data confidently across business units.  

The most common pain points K_Maker is designed to address:  
- **No single source of truth** -  — fragmented lists with conflicting definitions across reports, APIs and downstream systems
- **Limited version control or provenance** - ad-hoc spreadsheet management with no audit trail, change history or approval process
- **Brittle per-system mappings** - custom integrations that break whenever a reference list is updated, creating ongoing rework
- **No stable identifiers** — concepts with no persistent URI, making reliable cross-system linking impossible
- **High barrier to graph native adoption** - graph-native strategies are appealing but existing teams have no accessible entry point  

::KCard
Organisations increasingly recognise that graph-native data management is the future of enterprise data. Vocabulary management is the accessible, high-value entry point — delivering immediate governance wins while building the foundation for broader Knowledge Graph maturity.

## K-Maker's solution
K-Maker delivers four core capabilities that directly address the root causes of reference data fragmentation — without requiring ground-up software development or disruption to existing workflows.

::KCard
Replace fragmented lists with version-controlled, machine-readable vocabularies. Every concept has a stable, permanent identifier (IRI), a formal definition, and full provenance — so any system, report or team is always working from the same authoritative record.
#title
A governed single source of truth
::

::KCard
K-Maker UIs are built using SHACL (Shapes Constraint Language), ensuring every user interaction validates data against your specific graph at the point of entry. Invalid data cannot enter your vocabulary store.
#title
SHACL-validated data integrity
::

::KCard
Formalised change and approval processes via a purpose-built web UI — with staging review, version control, and granular, rol-based access. Every change is fully auditable from author edit through to production publication.
#title
Structured publishing workflows
::

::KCard
Any downstream system can consume governed vocabularies via REST API or embeddable web components — in multiple formats including JSON-LD, Turtle, CSV and RDF/XML. Integrate once; all consumers benefit from every future vocabulary update automatically.
#title
Seamless system integration
::
